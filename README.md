# Макет гибкой производственной линии

Интерактивный 3D-макет гибкой производственной линии с двумя стеллажами, зонами загрузки/выгрузки, модульным полем и внешним управлением через FastAPI.

Проект объединяет:

- клиент на `Three.js` и `Vite`
- backend на `FastAPI`
- Swagger/OpenAPI для ручного тестирования
- внешний Python-скрипт, который управляет стендом через API как отдельная программа управления

## Что умеет проект

- Визуализирует поле `6 x 9`, два боковых стеллажа, два крана и зоны загрузки/выгрузки.
- Позволяет вручную редактировать раскладку поля: конвейеры, поворотные модули и блоки обработки.
- После выключения режима редактирования переводит стенд в режим удаленного управления через API.
- Позволяет внешнему скрипту читать текущее состояние стенда и принимать решения на его основе.
- Поддерживает сценарий транспортировки объектов:
  - случайная раскладка объектов по стеллажам
  - перенос объектов в зону загрузки/выгрузки
  - движение объектов по полю по логике модулей
  - задержка на блоках обработки
  - поворот rotary-модулей через API
  - выгрузка объектов на противоположный стеллаж

## Режимы работы стенда

У стенда есть два главных режима.

### 1. Edit-режим

Используется для ручной настройки производственной линии.

Что доступно:

- выбор ячейки
- переключение типа модуля
- поворот модулей
- очистка ячеек

В этом режиме основная логика выполняется на клиенте.

### 2. API-режим

Используется для внешнего управления через FastAPI.

Что происходит при переходе:

- клиент синхронизирует текущую раскладку поля с backend
- локальные операционные действия блокируются
- клиент начинает подтягивать серверные снапшоты
- всеми объектами и краном теперь управляет сервер

Именно в этом режиме должен работать внешний сценарный Python-скрипт.

## Горячие клавиши

### Редактирование

- `ПКМ` по ячейке: выбрать ячейку
- `ЛКМ` по выбранной ячейке: сменить тип модуля
- `R`: повернуть выбранный модуль
- `Del`: очистить выбранную ячейку

### API-управление

- `Q`, `E`: выбрать активный кран
- `A`, `D`: перемещение крана по рядам
- `W`, `S`: перемещение крана по уровням
- `F`: забрать или положить объект на стеллаж
- `G`: забрать или положить объект в зону загрузки/выгрузки
- `Стрелки`: перемещение объекта по полю

## Модель производственного поля

На поле поддерживаются следующие типы модулей:

- `empty`
- `conveyor`
- `rotary`
- `process`

### Conveyor

Объект идет по направлению стрелки модуля.

### Rotary

Поворотный модуль может быть развернут в одно из четырех направлений.

Внешний скрипт может:

- прочитать текущее направление
- выбрать допустимый выход
- отправить команду на поворот модуля через API

### Process

Блок обработки моделирует кратковременную задержку объекта перед продолжением движения.

## Краткое описание endpoint'ов

### `GET /api/state`

Возвращает полный снапшот текущего состояния:

- режим
- активный кран
- активная зона загрузки/выгрузки
- состояние всех ячеек поля
- содержимое стеллажей
- содержимое IO-зон
- положение кранов
- все объекты

### `POST /api/mode`

Переключает сервер между `edit` и `api`.

Пример:

```json
{
  "edit_mode": false
}
```

### `POST /api/layout/sync`

Синхронизирует layout поля с сервером.

Используется клиентом при выходе из `edit`-режима.

### `POST /api/layout/rotary/set-direction`

Меняет направление поворотного модуля.

Пример:

```json
{
  "r": 2,
  "c": 3,
  "direction": 1
}
```

Где `direction`:

- `0` = north
- `1` = east
- `2` = south
- `3` = west

### `POST /api/scenario/randomize-warehouses`

Случайно раскладывает объекты по обоим стеллажам.

Пример:

```json
{
  "count_per_side": 18,
  "seed": 42
}
```

### `POST /api/crane/move`

Двигает активный кран.

Пример:

```json
{
  "direction": "right"
}
```

Допустимые значения:

- `left`
- `right`
- `up`
- `down`

### `POST /api/crane/rack-exchange`

Забирает объект со стеллажа или кладет его на стеллаж в зависимости от состояния активного крана.

### `POST /api/crane/io-exchange`

Забирает объект из зоны загрузки/выгрузки или кладет туда объект из активного крана.

### `POST /api/io/launch`

Помещает нижний объект активной зоны загрузки/выгрузки на поле.

### `POST /api/field/move`

Двигает активный объект по полю.

Пример:

```json
{
  "direction": "up"
}
```

## Внешний сценарный скрипт

Основной внешний сценарий расположен в:

- [scripts/run_api_scenario.py](/abs/path/c:/Users/ichiro/YandexDisk/work/csr/resresh-courses/2026/ИИ/gpl-virtual-stand/scripts/run_api_scenario.py)

### Идея скрипта

Это демонстрация того, как внешняя управляющая программа может:

- читать текущее состояние линии
- не отправлять объект в пустую ячейку
- следовать по направлению конвейеров
- задерживать объект на блоке обработки
- выбирать выход на rotary-модуле
- управлять кранами
- переносить объекты между стеллажом, IO и полем

### Что делает сценарий

После запуска скрипт:

1. Проверяет, что стенд уже переведен в `api`-режим.
2. Устанавливает объем зоны загрузки/выгрузки.
3. Случайно раскладывает объекты по стеллажам.
4. Левым краном переносит `4` объекта в левую зону загрузки/выгрузки.
5. По одному запускает объекты на поле.
6. Ведет объект через поле:
   - на `conveyor` идет по стрелке
   - на `process` делает паузу
   - на `rotary` выбирает допустимый выход, не ведущий назад
7. Доводит объект до противоположной зоны загрузки/выгрузки.
8. Правым краном переносит объекты на правый стеллаж.

### Запуск сценария

```powershell
python scripts/run_api_scenario.py --base-url http://127.0.0.1:8000
```

### Полезные параметры

```powershell
python scripts/run_api_scenario.py --seed 42 --count 4 --process-delay 1.0 --step-delay 0.2
```

Параметры:

- `--base-url` адрес FastAPI
- `--seed` seed для случайной раскладки и выбора выхода rotary
- `--count` сколько объектов транспортировать
- `--process-delay` пауза на блоке обработки
- `--step-delay` задержка между шагами API

## Типовые проблемы и решения

### Сценарий отправляет запросы, но на фронте ничего не видно

Проверьте:

- что выключен `edit`-режим
- что backend запущен
- что клиент открыт именно на `http://127.0.0.1:5173`

В текущей версии клиент обновляет сцену по серверным снапшотам в `api`-режиме.

### Swagger открывается, но команды не выполняются

Обычно причина одна из двух:

- стенд все еще находится в `edit`-режиме
- команда противоречит текущему состоянию, например кран не стоит у нужной зоны

### Внешний сценарий завершается ошибкой маршрута

Это значит, что на поле:

- нет корректного пути до противоположной IO-зоны
- есть развилка, которую скрипт не может разрешить однозначно
- путь ведет в пустую ячейку
- образуется цикл

Нужно поправить layout поля.

### `npm run dev` не запускается

Проверьте отдельно:

```powershell
npm run dev:client
npm run dev:server
```

Если сервер не стартует, убедитесь, что Python-зависимости установлены.
